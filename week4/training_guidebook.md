## 第一部分：技术叙事与工程定位（The Storyline）

在评估前沿人工智能（如状态空间世界模型）跨越 Sim-to-Real（虚实迁移）鸿沟的能力时，学术界缺乏包含 **"软材料迟滞"** 与 **"生物随机噪声"** 的实体基准。

本项目旨在将 **"液态金属微地形"** 与 **"活体生物运动"** 的耦合系统，打造为一个 **高复杂度的物理测试床（Testbed）**。

为证明该测试床的有效性，我们的工程技术分两步硬核求证：

1. **宏观因果量化（Statistical Ground Truth）**
   通过提取生物"奔跑-翻滚"动力学特征，证明微地形干预物理上真实有效。

2. **微观时序可学习性证明（Proof of Learnability）**
   通过训练一个 Baseline 神经网络，证明该系统产生的含噪数据依然具备确定性的隐藏动力学，能够被深度学习表征。

## 第二部分：数据清洗与统计初值设定（Data Processing & Statistics）

基于平台现有视觉算法输出特征（基础 5 Hz 检测 + 线性插值至 20 Hz），我们采用 **"空域与时域分流"** 的策略进行数据处理。

### 2.1 数据清洗分流规则

- **空域统计（如热力图）**
  对路径形貌保真度要求高，**使用全量 20 Hz 帧**（即所有 `found == true` 的行），以提升网格命中精度。

- **时域 / 微分统计（如转角、速度、停顿）**
  插值帧会严重抹平高频特征（制造 $0^\circ$ 转角伪影和微小伪速度）。**必须强制降采样，仅使用 `track_quality == 'observed'`（即真实 5 Hz）的数据帧** 进行相邻差分计算。

### 2.2 离散化统计参数（推荐初值配置）

为执行"奔跑-翻滚"模型的轨迹切分，算法组可直接采用以下经验初值进行第一轮迭代：

| 参数 | 推荐值 | 含义与逻辑 |
| :--- | :---: | :--- |
| **速度 / 停顿阈值** $v_{th}$ | $5.0\,\text{px/s}$ | 约为智能体身长的 $0.1$ 倍/秒。当 `observed` 帧间计算出的瞬时速度 $< 5.0\,\text{px}$ 时，判定为"停顿 / 等待（$\gamma$）"。 |
| **转角识别阈值** $\theta_{th}$ | $30^\circ$ | 过滤正常的触角探路摇晃。转弯角 $\lvert\Delta\theta\rvert > 30^\circ$ 且伴随降速时，记为一次有效重定向（Tumble）。 |
| **移动平均窗口** $N$ | $N = 3$ 个 `observed` 帧 | 约对应真实物理时间 $0.6\,\text{s}$。计算转角时不使用单帧相减，而是利用过去 3 个真实帧的向量滑动平均，可最大限度滤除视觉中心点漂移的抖动噪声。 |
| **空间网格划分** | $20 \times 20$ | 生成 Spatial Occupation Heatmap 时使用。 |

## 第三部分：基线预测模型架构与训练配置（Baseline Network）

该模型的核心任务是：在固定 **Flat** 条件下，基于过去 $T_{in}$ 秒的真实轨迹预测未来 $T_{out}$ 秒的轨迹。训练目标是建立正常行为动力学的 Baseline，并验证蚂蚁轨迹中可解释运动先验的可学习性。为契合世界模型主流趋势，采用轻量级 **Mini-Transformer** 架构。

> **训练范围约束**：训练、验证和测试只使用真实 Flat 数据。CornerUp / CornerDown 合成统计用于机制示意，不是监督数据，禁止送入模型。固定环境条件是基线实验的控制变量设计，使模型误差能够对应蚂蚁自身的时序随机性。

### 3.1 状态空间定义（State & Action Space）

- **输入状态向量（Input State $S_t$）**：维度为 5。

  $$
  S_t = \big[\,x,\ y,\ v,\ \cos(\theta),\ \sin(\theta)\,\big]
  $$

  > ⚠️ 强烈建议将角度 $\theta$ 拆解为 $\cos$ 和 $\sin$，以消除 $360^\circ \rightarrow 0^\circ$ 的突变数值回绕梯度问题。

- **环境上下文（Context）**：训练条件恒为 Flat。推荐直接省略地形动作向量；若为了保持通用接口而保留，则始终使用固定常量 `flat = [1]`，不可引入合成 Corner 标签。

### 3.2 神经网络架构参数（推荐初值）

| 参数 | 推荐值 | 说明 |
| :--- | :---: | :--- |
| **上下文历史窗口**（Context Length） | 输入过去 $10$ 个 `observed` 帧 | 代表过去 $2$ 秒的真实记忆 |
  | **预测未来步长**（Prediction Horizon） | 输出未来 $5$ 个 `observed` 帧 | 预测未来 $1$ 秒的轨迹 |
  | **数据划分** | 按 session 划分 70/15/15 | 禁止按帧随机划分，防止相邻窗口泄漏 |
| **骨干网络结构**（Architecture） | Transformer Encoder | — |
| 隐藏层维度 `d_model` | $128$ | — |
| 注意力头数 `n_heads` | $4$ | — |
| 编码器层数 `num_layers` | $3$ 层 | — |
| 参数量规模 | 约 $100$ 万以内 | 极度轻量，防过拟合 |

### 3.3 服务器租赁与训练超参数

- **硬件需求**
  租用单张 **NVIDIA RTX 3090 (24G) / 4090 / 4070Ti** 均可。

- **训练超参数（Training Hyperparameters）**

  | 项 | 推荐值 |
  | :--- | :--- |
  | Optimizer（优化器） | **AdamW**（Weight decay = $1\text{e-}4$） |
  | Learning Rate（学习率） | $3\text{e-}4$（搭配 Cosine Annealing 学习率衰减） |
  | Batch Size（批大小） | $64$ |
  | Loss Function（损失函数） | **MSE Loss**（均方误差，计算预测坐标与真实坐标的 L2 距离） |
  | Epochs（迭代次数） | $200$，并设置 Early Stopping（若 Validation Loss 连续 20 轮不下降则提前终止），通常 **1 小时内** 可完成训练 |

## 第四部分：实验举证与学术级指标验收（Evaluation Artifacts）

在答辩 PPT 或 GitHub 仓库中，必须展示以下标准化输出物，以形成闭环证据链：

### 1. 宏观量化对比图（证明环境干预有效）

- **20 × 20 空间热力图**：无隆起（Flat）vs 有隆起（Terrain）状态下的对比。
- **转角分布直方图（Turn Angle Distribution）**：展示平地上的正态单峰分布，如何被地形强制演化为 $\pm 90^\circ$ 附近的多峰（冠形）分布。

### 2. 基线模型训练凭证（证明微观数据可学习）

- **Loss 收敛曲线图**：截取 TensorBoard / WandB 中的 Training Loss 与 Validation Loss 曲线图。
- **模型快照（Checkpoints）**：留存 `best_model.pth` 权重文件。

### 3. 时序预测标准评价指标（确立测试基准分数）

在测试集上运行已收敛的模型，必须列出以下两个轨迹预测领域的通用黑话指标：

- **ADE（Average Displacement Error，平均位移误差）**
  预测出的未来 5 帧轨迹与真实轨迹 5 帧坐标的平均欧氏距离。

- **FDE（Final Displacement Error，终点位移误差）**
  预测出的第 5 帧（即 1 秒后）坐标与真实位置的距离。

> **答辩话术**：
> "我们的基线模型在该物理沙盒中取得了 ADE = X、FDE = Y 的底线成绩，未来更先进的世界模型应以此为 Benchmark 进行超越。"
